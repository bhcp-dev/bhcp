# In-session registered-evidence forward fixture

This deliberately small Rust repair isolates the evidence boundary rather than
task difficulty. The agent receives the canonical contract, a fixed typed
candidate, and three project-registered process adapters. Public evidence checks
the visible readiness function; oracle evidence checks the withheld readiness
function; change-policy evidence accepts only the exact focused final source.
Each producer validates the request's subject content reference and judges those
exact supplied bytes rather than substituting the fixture's ambient project path.

The agent can retain and inspect a canonical evidence bundle before its final
claim. The independent experiment controller still applies its own public, oracle,
and change-policy judges after the model stops. Agreement between those two paths,
not the simplicity of the edit, is the forward-test outcome.
