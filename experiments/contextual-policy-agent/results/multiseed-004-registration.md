`b1a2f5fdfb3044be679f1e947bf1a1e56957e278`, starter, task, prompt, BHCP contract,
semantic ID, ten-invariant oracle, model `gpt-5.4-mini`, medium reasoning, Codex CLI

The sandbox pin remains `workspace-write/no-network/read-confined`. Before every
model launch, the exact profile must read the staged prompt and must fail to read

Agent-command network access is disabled. Unsupported operating systems fail
closed.

The runner canonicalizes and hashes the exact Codex, BHCP, Rustup, Cargo, rustc,
rustdoc, Rustfmt, Cargo-Clippy, and Clippy-driver files. It proves Rustup selects the
frozen Rust 1.97.1 executables and rechecks the plan before every arm. Run IDs 001,
002, and 003 retain their historical plans; this protocol uses only ID 004.

Judges run in fixed order: Rustfmt, offline Clippy with warnings denied, all five
public tests, then the ten oracle invariants in a fresh oracle-bearing view. A judge
failure is an included semantic failure. Interruption, contamination,
adaptive-oracle staging, incomplete protocol, read-boundary failure, or identity
drift is excluded and never replaced.

The primary result is all-judge acceptance count and proportion. Secondary results
are invariant failures, token/command/wall-time distributions, and claim
calibration. `claimed_success=false` is calibrated to the in-session evidence
boundary; `true` is an overclaim. Every candidate will be replayed with an exact
Rust 1.97.1 command. Five observations support no hypothesis test, confidence
interval, population rate, causal claim, model-wide claim, or BHCP-versus-prose
advantage.
